# 给 Codex 的使用顺序

请严格按下面顺序给 Codex，不要一次只丢图片，也不要只丢需求。

## 第一步：先给 Codex 这份说明
告诉 Codex：

> 你要开发的是一个本地电脑端「美食养生贴图生成工具 V1」。
> 先不要做全量复杂功能，第一阶段只跑通核心链路：上传素材、视频预览截图、图片裁剪、文字上图、单图导出。
> 参考图只用于理解页面结构和交互，不要求逐像素照抄。
> 成本控制优先：第一版不接 AI API，不做云端，不接飞书，不做小红书/抖音爬虫。

## 第二步：给 Codex 总需求文档
文件：`01_PRD_美食养生贴图工具V1总需求.md`

让 Codex 先阅读，不要立刻写代码。阅读后让它输出：

1. 它理解的产品目标
2. 它计划的技术架构
3. 它准备先做的 5 个核心功能
4. 有哪些需求它认为需要澄清

如果它理解偏了，先纠正它，再让它开始。

## 第三步：给 Codex 开发提示词
文件：`02_Codex第一阶段开发提示词.md`

这个文件是第一阶段开工指令。不要让 Codex 一次做完整 V1，先让它做第一阶段。

第一阶段只验收这 5 个功能：

1. 上传视频/图片
2. 视频预览和手动截图
3. 图片裁剪
4. 文字上图
5. 单图导出

## 第四步：给 Codex 参考图
文件夹：`reference_images/`

包含 4 张图：

1. `01_上传素材与视频选图页.png`
2. `02_图片编辑与批量处理页.png`
3. `03_字体样式与背景边框设置页.png`
4. `04_拼图比例与导出页.png`

提醒 Codex：参考图是功能布局参考，不要求完全照抄 UI 样式。

## 第五步：等第一阶段跑通后，再给功能优先级表
文件：`03_功能优先级表.md`

让 Codex 逐步开发 P0 → P1 → P2，不要跳着做。

## 第六步：最后再加提示词模板
文件：`04_内置提示词模板20套.md`

第一阶段可以先把提示词模板做成一个本地 JSON/Markdown 配置，不接 API。助理复制到豆包/即梦使用。

## 最重要的限制

第一版不要做这些：

- 不做小红书/抖音自动采集
- 不做链接解析下载
- 不接飞书数据
- 不做云端多人协作
- 不接 AI 生图 API
- 不做 AI 判断哪张最爆
- 不做完整手机版
- 不做复杂账号后台

第一版的核心是：

> 本地工具，低成本，能选图、裁剪、提清晰、加字、导出。
