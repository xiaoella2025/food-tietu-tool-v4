### 2.1 直接图片上传入口（优先级 P0）

**问题：** 此前只能从视频截帧，没有直接上传图片的入口。

**实现：**

`app.js` — `renderVideoArea()` 中新增「🖼 上传图片」按钮，与视频上传按钮并列显示：

```html
<button id="btn-upload-images" style="...">🖼 上传图片</button>
```

`bindVideoPageEvents()` 中绑定事件：
```js
document.getElementById('btn-upload-images')?.addEventListener('click', triggerImageUpload);
```

`triggerImageUpload()` — 创建多选文件输入框，`accept="image/*" multiple`：
```js
function triggerImageUpload() {
  const input = document.createElement('input');
  input.type = 'file';
  input.accept = 'image/*';
  input.multiple = true;
  input.onchange = e => {
    const files = Array.from(e.target.files || []);
    files.forEach(loadImageFile);
  };
  input.click();
}
```

`loadImageFile()` — FileReader 读取图片，用 canvas 绘制（保持原始分辨率 `naturalWidth/naturalHeight`），生成 PNG dataUrl，构造成与视频截帧完全一致的 frame 对象结构，插入 `capturedFrames` 数组头部：

```js
function loadImageFile(file) {
  const reader = new FileReader();
  reader.onload = e => {
    const img = new Image();
    img.onload = () => {
      const canvas = document.createElement('canvas');
      canvas.width = img.naturalWidth;
      canvas.height = img.naturalHeight;
      canvas.getContext('2d').drawImage(img, 0, 0);
      const dataUrl = canvas.toDataURL('image/png');
      const frame = {
        id: `img-${Date.now()}-${Math.random().toString(36).slice(2, 7)}`,
        sourceDataUrl: dataUrl,
        versionsCache: {},
        activeRatio: 'original',
        time: file.name.replace(/\.[^.]+$/, '').slice(0, 12), // 文件名作为时间标签
        edited: false,
      };
      state.capturedFrames.unshift(frame);
      state.selectedFrameId = frame.id;
      updatePoolDOM(); // 不触发 full render，保持视频播放
      showToastDirect(`已加入图片：${file.name}`);
    };
    img.src = e.target.result;
  };
  reader.readAsDataURL(file);
}
```

**CSS 适配：** `.video-overlay` 从 `display:flex; justify-content:center` 改为 `flex-direction:column; gap:10px`，让两个按钮纵向排列；添加 `pointer-events: all` 使按钮可点击。

### 2.2 视频画面区域截图（优先级 P0）

**问题：** 需要支持在当前视频帧上框选一块区域，只截取该区域，而非整帧。

**状态变量（app.js）：**
```js
regionSelecting: false,   // 是否处于选区模式
regionRect: null,         // { sx, sy, sw, sh } — 视频像素坐标，不是显示坐标
```

**UI 入口：** `renderCaptureActions()` 新增「✂ 截取选区」按钮：
```html
<button id="btn-select-region" ${!state.videoUrl ? 'disabled' : ''}>✂ 截取选区</button>
```

**触发流程：**
1. 点击「截取选区」→ `startRegionSelection()` → `state.regionSelecting = true` → `render()`
2. `renderVideoArea()` 检测到 `regionSelecting = true`，渲染半透明遮罩层和确认/取消按钮：
```html
<div class="region-overlay" id="region-overlay">
  <div class="region-info">拖动框选截图区域</div>
  <button id="btn-confirm-region" class="primary">✓ 确认截取</button>
  <button id="btn-cancel-region">✕ 取消</button>
</div>
```

**拖拽实现（`bindRegionDrag`）：**
- `mousedown` → 记录起始点，创建 `.region-box` DOM 元素
- `mousemove`（document 级）→ 实时更新 box 的 left/top/width/height
- `mouseup` → 计算真实像素坐标映射：
```js
const scaleX = (video.videoWidth || 1920) / rect.width;   // rect = video-area 的 display rect
const scaleY = (video.videoHeight || 1080) / rect.height;
const sx = Math.round((boxRect.left - rect.left) * scaleX);
const sy = Math.round((boxRect.top - rect.top) * scaleY);
const sw = Math.round(boxRect.width * scaleX);
const sh = Math.round(boxRect.height * scaleY);
state.regionRect = { sx, sy, sw, sh };
```
- 尺寸过小（< 20px）则忽略
- 在选框上显示像素尺寸 badge

**截取（`captureRegion`）：**
```js
const { sx, sy, sw, sh } = state.regionRect;
const canvas = document.createElement('canvas');
canvas.width = sw;
canvas.height = sh;
canvas.getContext('2d').drawImage(video, sx, sy, sw, sh, 0, 0, sw, sh);
// → canvas.toDataURL → frame 对象，时间标签带 "选区" 后缀
```

**CSS 新增：**
```css
.region-overlay { position:absolute; inset:0; z-index:10; background:rgba(0,0,0,0.55); cursor:crosshair; }
.region-box { position:absolute; border:2px solid #fff; background:transparent; box-shadow:0 0 0 9999px rgba(0,0,0,0.4); }
```

### 2.3 左侧素材列表改为可展开树状结构（优先级 P1）

**问题：** 此前左侧图片列表是简单的平铺缩略图，看不到版本信息，无法切换到不同裁剪版本。

**新结构（`renderImageListItem`）：**

```
▼ 素材名 (展开时)
  ├ [原图缩略图] 原图
  ├ [3:4缩略图] 3:4
  └ [1:1缩略图] 1:1
```

HTML 结构：
```html
<div class="img-list-group" data-img-group-id="${frame.id}">
  <div class="img-list-row" data-img-row-id="${frame.id}">
    <div class="img-list-thumb"><img src="${frame.sourceDataUrl}"></div>
    <div class="img-list-info">
      <div class="img-list-name">${frame.time}</div>
      <div class="img-list-meta">
        <span class="ver-count">${versions.length}版本</span>
        <span class="ver-count edited">已编辑</span>
      </div>
    </div>
    <div class="img-list-expand">▼</div>
  </div>
  <div class="img-list-versions">
    <div class="ver-item source-ver" data-ver-src="${frame.id}">
      <span class="ver-thumb"><img src="${frame.sourceDataUrl}"></span>
      <span class="ver-label">原图</span>
    </div>
    ${versions.map(ratio => `
      <div class="ver-item" data-ver-ratio="${ratio}" data-ver-frame="${frame.id}">
        <span class="ver-thumb"><img src="${frame.versionsCache[ratio]}"></span>
        <span class="ver-label">${ratio}</span>
      </div>
    `).join('')}
  </div>
</div>
```

**事件绑定（两段 click）：**
- `.img-list-row` 点击 → 切换当前编辑对象
- `.img-list-expand`（▶/▼ 箭头）点击 → toggle `.expanded` class，展开/折叠版本列表
- `.ver-item`（版本项）点击 → 切换到该版本的编辑状态

**CSS 新增（tree UI）：**

### 2.4 确认既有实现的正确性（优先级 P1）

通过代码审查，确认以下设计已正确实现，无需修改：

#### 裁剪已支持用户拖动（非自动裁剪）
- `ImageEdit.js` 中 `onCropMouseDown / onCropMouseMove / onCropMouseUp` 处理拖动
- `hitTestCropCorner()` 检测 8 个控制点（4角 + 4边）+ 框内移动
- `resizeCrop()` 根据 handle 类型和 `crop.ratio`（锁定比例）调整尺寸
- 初始裁剪框由 `fitCropToRatio()` 计算（居中，最大化尺寸）
- 用户可拖动位置，确认后才 `saveCropAsVersion()` 生成版本

#### 非破坏性版本管理已正确
- `sourceDataUrl` 永远不被修改
- `versionsCache` 字典存储各比例的 dataUrl
- `switchToRatio()` 切换时：若有缓存则显示缓存；无缓存则从 `sourceImageEl` 重新生成
- `generateCropPreviewFromSource()` — 裁剪从原始分辨率（`sourceImageEl.width/height`）进行，而非预览 canvas 尺寸

#### 截图使用原始视频分辨率
- `captureCurrentFrame()` 使用 `video.videoWidth` 和 `video.videoHeight`（非 `clientWidth/clientHeight`）
- `captureRegion()` 同理，使用视频像素坐标而非显示坐标

#### 截图池竖向多行布局
- `.pool-grid { display:grid; grid-template-columns: repeat(auto-fill, minmax(120px, 1fr)); }`
- 最多 24 张，超出后 oldest 被移除

#### 截图后视频时间不跳回
- `captureCurrentFrame()` 调用 `updatePoolDOM()`（仅更新 pool DOM） + `showToastDirect()`（不触发 render）
- 完整 render 只在首次加载视频时调用

## 四、非破坏性编辑架构（关键设计说明）

```
capturedFrames[frameId]
  ├── sourceDataUrl          ← 永远不变（截取/导入时的原始 dataUrl）
  ├── versionsCache:{}      ← { '3:4': dataUrl, '2.35:1': dataUrl, ... }
  ├── activeRatio: 'original' | '3:4' | '2.35:1' | ...
  └── edited: boolean

切换图片/版本 → 从 sourceDataUrl 或 versionsCache[ratio] 加载到 editCanvas
生成新版本 → 从 sourceImageEl（原始分辨率）重新裁剪/滤镜 → 存入 versionsCache
```
